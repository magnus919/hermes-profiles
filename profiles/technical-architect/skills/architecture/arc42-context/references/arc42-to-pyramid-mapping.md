# arc42-to-Pyramid Mapping

## Mapping Table

| arc42 Section | Pyramid Layer | What Lives There | Consumer |
|--------------|--------------|------------------|----------|
| Canvas / Quality tree | **L1** (01-summary/) | Top quality goals, key stakeholders, architecture overview | Any stakeholder |
| Section 1: Mission & Goals | **L2** (02-analysis/) | Business goals, functional requirements | Product managers, engineers |
| Section 2: Constraints | **L2** (02-analysis/) | Technical, organizational, regulatory constraints | Architects, engineers |
| Section 3: Context | **L2** (02-analysis/) | System context, external interfaces | Integrators |
| Section 4: Solution Strategy | **L2** (02-analysis/) | Architecture approach, key decisions | Architects, reviewers |
| Sections 5-8: Architecture | **L3** (03-dossiers/) | Detailed architecture views, patterns | Implementers |
| Section 9: Risks | **L3** (03-dossiers/) | Technical risks, mitigation strategies | Project managers |
| Section 10: Glossary | **L3** (03-dossiers/) | Terms and definitions | All consumers |

## Modes

arc42 supports three depth modes that map to pyramid coverage:
- **Lean (L1 only):** Canvas + quality tree only
- **Essential (L1 + L2):** Sections 1-4
- **Thorough (all layers):** Full template
