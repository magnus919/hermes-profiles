# Proofreading Protocol

## The Final Gate

Proofreading is the last check before publication. After proofreading, the piece goes to the verifier for the publication gate.

## What Proofreading Covers

- Spelling errors (including homophones: there/their/they're)
- Missing or doubled words
- Punctuation errors (missing periods, unbalanced quotes)
- Formatting artifacts (extra spaces, line break issues)
- Character encoding issues (smart quotes vs straight quotes)
- Page layout issues (widows, orphans, bad breaks)

## What Proofreading Does NOT Cover

- Argument quality (editor's domain)
- Factual accuracy (fact-checker's domain)
- Voice consistency (editor's domain)
- Structural issues (editor's domain)

## Protocol

1. Read the piece aloud — this catches errors the eye skips over
2. Read backward (last paragraph first) — this breaks pattern recognition and catches spelling errors
3. Check every link by following it
4. Check every image and caption
5. Verify formatting renders correctly (headings, lists, code blocks)
