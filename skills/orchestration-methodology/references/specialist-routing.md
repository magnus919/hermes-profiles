# Specialist Routing

## The Routing Rule

Given the same set of specialists, ordering them differently produces completely different outcomes.

## Routing Patterns

| Pattern | Order | Use When |
|---------|-------|----------|
| **Investigate → Decide** | Researcher → Data Architect → Product Manager | The question needs evidence gathering before decision-making |
| **Design → Build** | Technical Architect → Implementation Planner | Architecture must be defined before work can be sequenced |
| **Create → Polish** | Writer → Editor → Copy Editor → Verifier | Content pipeline with progressive refinement |
| **Explore → Converge** | Multiple specialists in parallel → Orchestrator synthesis | The question benefits from diverse simultaneous perspectives |

## Routing Decisions

For each sub-task:
1. Which profile has the right methodology?
2. Is this profile available or blocked?
3. What does this profile need from previous profiles?
4. What will the next profile need from this one?
