# User Research Methods

## Choosing a Method

| Method | When to Use | Output |
|--------|-------------|--------|
| **Contextual inquiry** | Early discovery — understand how users currently work | Task flows, pain points, workarounds |
| **Structured interview** | Exploratory — understand needs, motivations, mental models | Themes, quotes, journey maps |
| **Usability test** | Evaluate a design — find where users struggle | Task success rates, error patterns, satisfaction |
| **Survey** | Quantitative — measure satisfaction, usage patterns at scale | Statistics, distributions, correlations |
| **Card sorting** | Information architecture — how users categorize content | Category structures, labels |
| **A/B test** | Compare two designs — which performs better on a metric | Statistical comparison, effect size |

## Research Protocol

1. **Define the research question.** What decision will this research inform?
2. **Recruit participants.** 5 users per segment finds 85% of usability issues.
3. **Prepare a discussion guide.** Script the introduction, tasks, and follow-ups.
4. **Run the sessions.** Observe, don't lead. Ask "why" after behaviors.
5. **Synthesize findings.** Affinity mapping, theme extraction, severity ratings.
